Code Talk

A better way to talk about code

No more hunting code

When you're writing about code, you're always trying to point to something specific. This can lead to the reader scrolling up and down to try to find what you're talking about and even if they didn't have to scroll, its a lot of effort!

No more unapproachable codebases

Need to get someone up to speed on a codebase? Give them a guided tour of the important parts of your code.

No more dead documentation

Instead of copy-pasting your code into a document, you can reference your code and get a notificaiton if the code you're referencing changes. You'll have living docs, that are closely tied to the code that they attempt to document.
